### Módulo 2 — Takeover Manual

**RF-007** ← RN-DA-032
O sistema deve sinalizar automaticamente para revisão toda interação cuja confiança esteja abaixo do threshold configurado (padrão: 80%). Interações sinalizadas habilitam o botão de takeover no painel. Interações com confiança acima do threshold mantêm o botão de takeover disponível para acionamento manual pelo Admin por qualquer motivo.

**Critério de aceite RF-007:**
- Dado: Agente gera resposta com confiança < threshold configurado.
- Quando: A resposta é registrada.
- Então: Interação passa automaticamente para o estado "Sinalizada para revisão". Botão "Assumir conversa" é habilitado. Badge de status "Sinalizada para revisão" aparece na lista de interações.

---

**RF-008** ← RN-DA-033
O Admin deve poder iniciar takeover de qualquer interação (sinalizada ou não) clicando em "Assumir conversa". O sistema deve:
1. Registrar o takeover com: timestamp, motivo e Admin ID.
2. Exibir ao usuário: "Um analista da equipe Repasse Seguro assumiu essa conversa para ajudá-lo. Como posso ajudar?" — com avatar diferenciado (ícone de pessoa), nome "Equipe Repasse Seguro" e separador visual "Atendimento humano".
3. Parar imediatamente todas as respostas automáticas do agente para aquela sessão.
4. Manter o campo de entrada do usuário ativo normalmente.

**Critério de aceite RF-008:**
- Dado: Admin clica em "Assumir conversa" em uma interação.
- Quando: O takeover é confirmado.
- Então: Interação passa para "Em takeover" no painel. Usuário recebe a mensagem de takeover com avatar diferenciado e separador visual. Agente para de responder para aquela sessão. Log de takeover registrado com Admin ID, timestamp e motivo.

---

**RF-009** ← RN-DA-033
Quando o Admin encerrar um takeover, o agente deve retomar o controle da conversa. O usuário deve receber: "Você está novamente em atendimento com [nome do agente]." — com separador visual de retorno e avatar padrão do agente.

**Critério de aceite RF-009:**
- Dado: Admin clica em "Encerrar atendimento humano".
- Quando: O encerramento é confirmado.
- Então: Interação passa de "Em takeover" para estado anterior (RespondidaPelaIA ou SinalizadaParaRevisao dependendo da confiança). Usuário recebe mensagem de retorno do agente com avatar padrão e separador visual.

---

**RF-010** ← RN-DA-033
O sistema deve bloquear tentativa de takeover de conversa já encerrada. Admin recebe status "Conversa encerrada" ao tentar assumir.

**Critério de aceite RF-010:**
- Dado: Conversa está no estado "Encerrada".
- Quando: Admin tenta clicar em "Assumir conversa".
- Então: Botão exibe tooltip ou mensagem "Conversa encerrada". Nenhuma ação de takeover é registrada.

---

**RF-011** ← RN-DA-033
Quando dois Admins tentarem fazer takeover da mesma conversa simultaneamente, o primeiro a confirmar deve assumir. O segundo Admin deve receber: "Esta conversa já está em atendimento por outro analista."

**Critério de aceite RF-011:**
- Dado: Dois Admins clicam em "Assumir conversa" da mesma interação no mesmo momento.
- Quando: Ambas as solicitações chegam ao backend.
- Então: Lock otimista garante que apenas o primeiro seja confirmado. O segundo recebe mensagem de conflito. Nenhuma inconsistência de estado é registrada.

### Módulo 3 — Dashboard de Métricas

**RF-012** ← RN-DA-034
O sistema deve exibir um Dashboard de Métricas no painel de Supervisão IA com os seguintes indicadores, filtrável por período (dia, semana, mês) e por agente (Dani-Cessionário, Dani-Cedente):
- Volume de interações: total de mensagens por dia e semana.
- Top 10 perguntas mais frequentes no período.
- Taxa de respostas com recusa: percentual de respostas recusadas por restrição de perfil.
- CSAT médio: média das avaliações de satisfação no período (escala 1–5).
- Tempo médio de resposta: latência média por tipo de interação.

**Critério de aceite RF-012:**
- Dado: Admin acessa o Dashboard de Métricas.
- Quando: Dados estão disponíveis para o período selecionado.
- Então: Todos os 5 indicadores são exibidos com valores corretos. Filtros por período e agente funcionam independentemente. Métricas atualizam em tempo real conforme novas interações ocorrem.

---

**RF-013** ← RN-DA-034
Quando um indicador não tiver dados suficientes no período selecionado, o sistema deve exibir "Dados insuficientes para o período selecionado" para aquele indicador específico, com ícone de informação. O card mantém sua estrutura visual. O sistema nunca exibe "0" ou "0%" para indicadores sem dados.

**Critério de aceite RF-013:**
- Dado: Admin seleciona um período sem dados para um ou mais indicadores.
- Quando: O Dashboard é renderizado.
- Então: Card do indicador sem dados exibe "Dados insuficientes para o período selecionado" + ícone de informação. Estrutura do card é mantida (título, layout). Zero ou 0% nunca são exibidos como valor para indicador sem dados.
