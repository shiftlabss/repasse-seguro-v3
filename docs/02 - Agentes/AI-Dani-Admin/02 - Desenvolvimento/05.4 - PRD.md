### Módulo 7 — Controle de Acesso

**RF-026** ← RN-DA-030, RN-DA-031, RN-DA-032, RN-DA-033, RN-DA-034, RN-DA-035
O sistema deve aplicar a seguinte matriz de permissões em todos os endpoints e telas do módulo AI-Dani-Admin:

| Operação | Admin | Cessionário | Cedente |
|---|---|---|---|
| Acessar painel de Supervisão IA | ✅ Permitido | ❌ Bloqueado (403) | ❌ Bloqueado (403) |
| Visualizar interações de qualquer usuário | ✅ Permitido | ❌ Apenas próprias | ❌ Apenas próprias |
| Iniciar takeover | ✅ Permitido | ❌ Não se aplica | ❌ Não se aplica |
| Configurar threshold | ✅ Permitido | ❌ Bloqueado (403) | ❌ Bloqueado (403) |
| Visualizar Dashboard de métricas | ✅ Permitido | ❌ Bloqueado (403) | ❌ Bloqueado (403) |
| Reativar agente após desligamento | ✅ Permitido | ❌ Bloqueado (403) | ❌ Bloqueado (403) |
| Receber alertas automáticos | ✅ Via Slack e e-mail | ❌ Não se aplica | ❌ Não se aplica |
| Ajustar rate limit do webchat | ✅ Permitido | ❌ Bloqueado (403) | ❌ Bloqueado (403) |
| Visualizar logs de auditoria | ✅ Permitido | ❌ Bloqueado (403) | ❌ Bloqueado (403) |

**Critério de aceite RF-026:**
- Dado: Usuário com role diferente de Admin tenta acessar qualquer endpoint do módulo AI-Dani-Admin.
- Quando: A requisição é processada.
- Então: Resposta 403 Forbidden retornada. Nenhum dado do módulo Admin é exposto. Tentativa registrada em log de segurança.

## 3. Requisitos Não Funcionais

**RNF-001 — Performance**
- API do painel de Supervisão IA: resposta < 500ms no p95 para listagem de interações com até 1.000 registros.
- Dashboard de métricas: carregamento inicial < 1.5s. Atualização em tempo real via Supabase Realtime com latência máxima de 2s.
- Takeover: confirmação e aplicação do lock < 300ms para prevenir conflito simultâneo (RF-011).

**RNF-002 — Disponibilidade**
- Painel Admin: disponibilidade ≥ 99,5% (SLA mensal).
- Alertas automáticos: disparados em ≤ 60 segundos após detecção da condição crítica.
- Sistema de alerta não deve depender exclusivamente da disponibilidade da API do modelo de IA — o monitoramento opera mesmo quando o modelo está em FallbackAtivo.

**RNF-003 — Segurança**
- Todos os endpoints do módulo Admin protegidos com JWT válido e role `ADMIN` verificado.
- Dados de usuários nos logs de interação anonimizados por padrão — nome/CPF nunca exibidos na listagem.
- Logs de auditoria de ações Admin (takeover, threshold) imutáveis após criação — sem possibilidade de deleção ou edição retroativa.
- Secrets e API keys exclusivamente em variáveis de ambiente. Nenhuma credencial no código-fonte.
- HTTPS obrigatório em todos os ambientes exceto localhost.

**RNF-004 — Compliance (LGPD)**
- Dados de interação armazenados com identificadores anonimizados.
- Retenção máxima de histórico de interações: 90 dias (RN-DA-036). Deleção automatizada após janela de retenção.
- Logs de auditoria de ações Admin: retenção de 1 ano (por exigência de auditoria interna).
- Dados pessoais identificáveis visíveis apenas na visualização individual de interação, acessível exclusivamente a Admin autenticado.

**RNF-005 — Rastreabilidade**
- Toda ação Admin (takeover, encerramento de takeover, alteração de threshold, reativação de agente) deve ser registrada em log de auditoria com: Admin ID, tipo de ação, timestamp ISO 8601, dados relevantes (threshold anterior/novo, motivo de takeover).
- Logs de acesso ao painel: Admin ID + timestamp + filtros aplicados.

**RNF-006 — Escalabilidade**
- Sistema de alertas baseado em filas RabbitMQ com retry e dead-letter queue — suporta pico de múltiplos alertas simultâneos sem perda.
- Cache Redis para métricas agregadas do dashboard — TTL 60s para reduzir carga no banco durante picos de acesso Admin.

**RNF-007 — Observabilidade**
- Langfuse obrigatório para tracing de todas as interações dos agentes supervisionados: custo, latência, qualidade, nível de confiança.
- Sentry obrigatório para error tracking de infraestrutura com alertas configurados para erros P0.
- PostHog para analytics de uso do painel Admin (eventos definidos em D02 - Stacks).

**RNF-008 — Acessibilidade**
- Interface do painel Admin atende WCAG 2.1 AA: contraste mínimo 4.5:1, navegação por teclado, atributos ARIA em componentes customizados.
- Todo estado de loading usa skeleton (conforme D04 - Motion Spec). Spinner proibido como substituto de conteúdo.

**RNF-009 — Rate Limiting**
- Webchat: 30 mensagens/hora por usuário (janela deslizante) — configurável pelo Admin.
- Endpoints Admin de configuração: máximo 10 alterações de threshold por hora por Admin — protege contra automação de configuração.

**RNF-010 — Resiliência**
- Falha no sistema de alertas (Slack/e-mail) não deve impactar a operação do agente nem do painel Admin.
- Falha no Langfuse não deve impactar a operação do agente — tracing é async e não-bloqueante.
- Desligamento automático do agente (RF-005) deve ser executado mesmo em caso de falha parcial de outros serviços.

**RNF-011 — Testes de Pré-Lançamento**
- Filtro de escopo testado com 100% de sucesso em cenários de acesso cruzado antes de qualquer ativação em produção (RF-020).
- Mínimo 20 perguntas adversariais aprovadas antes do lançamento de instruções do agente (RF-023).
- Todos os 4 componentes de supervisão verificados como operacionais antes do lançamento (RF-024).

**RNF-012 — Concorrência**
- Takeover simultâneo por dois Admins na mesma conversa deve ser resolvido por lock otimista — garantia de consistência sem condição de corrida (RF-011).
- Dashboard de métricas suporta até 10 Admins simultâneos sem degradação perceptível de performance.
