> **TL;DR**
>
> - **Equipe:** ciclo de vida completo de usuários internos (convidar, ativar, editar papel, suspender, reativar, desligar, transferir Casos) — responsabilidade exclusiva do Admin RS. Desligamento bloqueia até redistribuição total de Casos. Dados históricos preservados para auditoria.
> - **Dashboard Executivo:** visão consolidada de pipeline e SLA para Coordenador RS e Admin RS (Analista RS não acessa). Aba "Financeiro" exclusiva do Admin RS com métricas de receita, margem e breakeven. Análise de funil de conversão com drill-down por estado.
> - **Configurações:** 14 parâmetros configuráveis pelo Admin RS. Parâmetros críticos requerem aprovação de dois Admins (quatro olhos). Log de auditoria imutável de todas as alterações. Gestão de versões de templates.
> - **Relatórios:** matrix de acesso por papel, exportação CSV/XLSX com log obrigatório para dados pessoais, relatório de inteligência de mercado anonimizado.
> - **Regra inegociável:** toda alteração em Configurações do Sistema gera registro de auditoria com usuário, data/hora, campo alterado e valores anterior/posterior.

## 1. Módulo: Equipe

### RF-CRM-060 — Ciclo de vida de usuários internos
**Origem:** RN-139
**Papel:** Admin RS

O sistema deve:
- Disponibilizar ao Admin RS as operações completas de ciclo de vida: convidar, ativar, editar papel, suspender, reativar e desligar usuários internos.
- Manter os seguintes estados possíveis para usuários:
  - **Ativo:** acesso pleno conforme papel.
  - **Suspenso:** acesso temporariamente bloqueado; Casos permanecem atribuídos ao usuário suspenso.
  - **Desligado:** acesso permanentemente revogado; Casos redistribuídos antes do desligamento.
- Respeitar as transições de estado: Ativo → Suspenso → Ativo (reativação) · Ativo → Desligado · Suspenso → Desligado. Desligado é estado terminal.

**Critério de aceite:** Somente o perfil Admin RS vê as opções de suspender/reativar/desligar no perfil de outro usuário. Tentativa de desligar usuário com Casos ativos não redistribuídos é bloqueada. Estado "Desligado" não tem opção de reverter para "Ativo".

---

### RF-CRM-061 — Suspensão de usuário interno
**Origem:** RN-140
**Papel:** Admin RS

O sistema deve:
- Exibir ao Admin RS o fluxo "Equipe → perfil do usuário → Suspender acesso".
- Exigir motivo da suspensão (selecionável: Férias · Afastamento médico · Investigação interna · Outros).
- Se o usuário tem Casos ativos: exibir lista dos Casos e perguntar "Deseja redistribuir os Casos agora ou manter atribuídos ao usuário suspenso?".
- Após confirmação da suspensão: bloquear login imediatamente, encerrar todas as sessões ativas do usuário e enviar e-mail informativo ao usuário: "Seu acesso ao CRM foi temporariamente suspenso. Entre em contato com seu gestor para mais informações."

**Critério de aceite:** Usuário suspenso não consegue logar imediatamente após a suspensão (sessão ativa encerrada). E-mail informativo enviado. Log de auditoria registra motivo da suspensão, Admin RS responsável e timestamp.

---

### RF-CRM-062 — Desligamento de usuário interno
**Origem:** RN-141
**Papel:** Admin RS

O sistema deve:
- Bloquear o desligamento se o usuário ainda tem Casos ativos não redistribuídos — exibir lista dos Casos pendentes de redistribuição.
- Após redistribuição de todos os Casos: confirmar desligamento, revogar acesso permanentemente e encerrar todas as sessões ativas.
- Preservar todos os dados históricos do usuário desligado (Atividades registradas, Casos conduzidos, linha do tempo) — nunca excluir. Exibir nome nos registros históricos como "[Nome] (Desligado)".
- Bloquear reutilização do e-mail do usuário desligado para criação de nova conta no CRM por 180 dias.

**Critério de aceite:** Tentativa de desligar usuário com 3 Casos ativos bloqueia e lista os Casos. Após redistribuição, desligamento concluído. Atividades antigas do usuário desligado aparecem na linha do tempo com "[Nome] (Desligado)". Tentativa de criar nova conta com o mesmo e-mail em menos de 180 dias é bloqueada.

---

### RF-CRM-063 — Alteração de papel de usuário interno
**Origem:** RN-142
**Papel:** Admin RS

O sistema deve:
- Permitir ao Admin RS alterar o papel de um usuário entre Analista RS e Coordenador RS.
- Ao rebaixar de Coordenador RS para Analista RS: verificar se o usuário tem Casos de outros Analistas sob sua supervisão e exibir alerta para redistribuição.
- Ao promover de Analista RS para Coordenador RS: conceder acesso imediato a todos os Casos da equipe e ao Dashboard consolidado.
- Registrar a alteração de papel no log de auditoria com: data, Admin RS responsável, papéis anterior e posterior.
- Bloquear a criação de novos usuários com papel Admin RS sem aprovação de outro Admin RS existente (quatro olhos).

**Critério de aceite:** Promoção a Coordenador RS dá acesso ao Dashboard em ≤1 minuto. Criação de Admin RS sem aprovação de segundo Admin é bloqueada. Log de auditoria registra a mudança de papel com todos os campos obrigatórios.

---

### RF-CRM-064 — Relatório de carga da equipe
**Origem:** RN-143
**Papel:** Coordenador RS, Admin RS

O sistema deve:
- Exibir em "Equipe → Carga atual" a distribuição de Casos entre os Analistas RS, com as seguintes métricas por Analista:
  1. Total de Casos ativos por estado.
  2. Casos com SLA em risco (amarelo) e vencido (vermelho).
  3. Follow-ups vencidos sem resolução.
  4. Tempo médio de conclusão dos últimos 10 Casos fechados.

**Critério de aceite:** Analista RS não acessa "Equipe → Carga atual". Dados por Analista exibem os 4 itens especificados. Atualização dos dados em ≤5 minutos.

## 2. Módulo: Dashboard Executivo

### RF-CRM-065 — Visão executiva do Pipeline
**Origem:** RN-144
**Papel:** Coordenador RS, Admin RS

O sistema deve:
- Exibir o Dashboard Executivo com as seguintes métricas em tempo real (≤5 min de defasagem):

| Métrica | Descrição |
|---|---|
| Casos abertos | Total por estado e por Analista RS |
| Taxa de conversão por estado | % de Casos que avançam de cada estado |
| Ciclo médio total | Dias corridos da abertura ao Fechamento (últimos 90 dias) |
| Ciclo médio por estado | Dias úteis médios em cada estado (últimos 90 dias) |
| Casos em risco de SLA | Alertas amarelos e vermelhos ativos |
| Previsão de Fechamentos | Casos em Formalização/Anuência com data estimada |
| Taxa de cancelamento | % de Casos cancelados por motivo (últimos 90 dias) |

- Bloquear acesso ao Dashboard Executivo pelo Analista RS — apenas Coordenador RS e Admin RS.

**Critério de aceite:** Analista RS não vê o item "Dashboard Executivo" na navegação. Métricas atualizam em ≤5 minutos após mudança de estado. Cada métrica exibe período de referência visível.

---

### RF-CRM-066 — Métricas financeiras do Dashboard (aba Financeiro)
**Origem:** RN-145
**Papel:** Admin RS exclusivo

O sistema deve:
- Exibir aba "Financeiro" no Dashboard exclusivamente para o Admin RS, com as seguintes métricas:

| Métrica | Descrição |
|---|---|
| Receita confirmada no mês | Soma das comissões de Casos Concluídos no mês corrente |
| Receita estimada no pipeline | Soma das comissões estimadas de Casos em Negociação/Anuência/Formalização |
| Ticket médio por Caso | Média de comissão total (cedente + cessionário) dos últimos 12 meses |
| Margem de contribuição estimada | Receita confirmada − custo incremental estimado por Caso |
| Descontos concedidos no mês | Soma e percentual médio de descontos aplicados |
| Casos no breakeven (≥3/mês) | Indicador binário: mês corrente atingiu ou não o breakeven operacional |

- Bloquear acesso à aba "Financeiro" para o Coordenador RS.

**Critério de aceite:** Coordenador RS não vê aba "Financeiro" no Dashboard. Admin RS vê todas as 6 métricas financeiras. Indicador de breakeven exibe "Atingido" ou "Não atingido" de forma visualmente clara.

---

### RF-CRM-067 — Análise de funil de conversão
**Origem:** RN-146
**Papel:** Coordenador RS, Admin RS

O sistema deve:
- Exibir em "Dashboard → Funil" a taxa de conversão entre estados, com:
  - Para cada par de estados consecutivos: percentual de Casos que avançaram vs. percentual que cancelaram.
  - Comparativo entre o mês corrente e a média histórica dos últimos 90 dias.
- Destacar em vermelho como "Gargalo identificado" estados com taxa de cancelamento acima de 20%.
- Ao clicar em qualquer estado do funil: exibir a lista dos Casos que cancelaram naquele estado com os respectivos motivos de cancelamento.

**Critério de aceite:** Estado com 25% de cancelamento exibe badge "Gargalo identificado" em vermelho. Estado com 15% de cancelamento não exibe o badge. Drill-down ao clicar abre lista de Casos cancelados com motivos.

## 3. Módulo: Configurações do Sistema

### RF-CRM-068 — Parâmetros configuráveis do CRM
**Origem:** RN-147
**Papel:** Admin RS

O sistema deve:
- Exibir em "Configurações → Parâmetros" os seguintes 14 parâmetros configuráveis:

| Parâmetro | Padrão |
|---|---|
| SLA por estado (dias úteis) | Tabela 5.1 (01.1) |
| Limite de Casos por Analista RS | 30 |
| Prazo de validade de proposta (horas) | 48h |
| Prazo de convite de Parceiro Externo (dias) | 7 dias |
| Prazo de arquivamento de Contato inativo (meses) | 12 meses |
| Prazo de arquivamento de Caso cancelado (dias) | 90 dias |
| Tabela de limites de desconto | Tabela RN-052 |
| Ciclo total alerta (dias corridos) | 60 dias |
| Follow-up: prazo de aviso (minutos antes) | 30 min |
| Inatividade de sessão (minutos) | 60 min |
| Retenção de dados — Casos cancelados (anos) | 5 anos |
| Retenção de dados — Casos concluídos (anos) | 10 anos |
| Hora de disparo do relatório semanal | Domingo 20h |
| Defasagem máxima de atividade retroativa (dias) | 30 dias |

- Gerar registro de auditoria automático para cada alteração: usuário, data/hora, campo alterado, valor anterior, valor novo.

**Critério de aceite:** Apenas Admin RS acessa "Configurações → Parâmetros". Toda alteração — mesmo de parâmetros não críticos — gera registro no log de auditoria. Parâmetros exibem o valor atual e o padrão de fábrica lado a lado.

---

### RF-CRM-069 — Processo de alteração de parâmetro crítico (quatro olhos)
**Origem:** RN-148
**Papel:** Admin RS (dupla aprovação)

O sistema deve:
- Classificar como parâmetros críticos: SLA por estado, limites de desconto, retenção de dados e limite de Casos por Analista.
- Para parâmetros críticos: exigir confirmação de segundo Admin RS (quatro olhos) antes de aplicar a alteração.
- Se há apenas 1 Admin RS ativo: exibir aviso "Este parâmetro requer aprovação de dois Admins. O sistema não tem um segundo Admin ativo. A alteração fica pendente de aprovação." Manter o valor anterior até aprovação.
- Após aprovação do segundo Admin RS: aplicar a alteração imediatamente e registrar no log de auditoria com os dois Admins envolvidos.

**Critério de aceite:** Alteração de SLA por estado por um único Admin RS cria aprovação pendente. Segundo Admin RS recebe notificação de aprovação pendente. Alteração só aplicada após aprovação de ambos. Log registra os dois responsáveis.

---

### RF-CRM-070 — Gestão de templates de mensagem
**Origem:** RN-149
**Papel:** Admin RS (criar/editar/desativar), Coordenador RS (aprovar)

O sistema deve:
- Permitir ao Admin RS criar, editar e desativar templates de mensagem.
- Templates ativos: disponíveis para todos os Analistas RS.
- Templates desativados: não aparecem para Analistas RS; mantêm-se no histórico para auditoria.
- Toda edição de template gera nova versão (v1.0, v1.1, v2.0). A versão anterior é arquivada — nunca excluída.
- Novos templates criados pelo Admin RS ficam disponíveis após aprovação do Coordenador RS.

**Critério de aceite:** Template desativado não aparece na lista de templates do Analista RS. Edição de template v1.0 gera v1.1 e arquiva v1.0 (acessível em "Histórico de versões"). Template novo em status "Aguardando aprovação" não está disponível para Analistas RS.

## 4. Módulo: Relatórios e Exportações

### RF-CRM-071 — Matrix de acesso a relatórios por papel
**Origem:** RN-150
**Papel:** Admin RS, Coordenador RS, Analista RS

O sistema deve:
- Disponibilizar os relatórios conforme a matrix de acesso abaixo:

| Relatório | Admin RS | Coordenador RS | Analista RS |
|---|---|---|---|
| Pipeline completo | Sim (todos) | Sim (equipe) | Sim (próprios) |
| Desempenho por Analista RS | Sim | Sim (equipe) | Não |
| Comissões confirmadas (período) | Sim | Não | Não |
| Comissões estimadas no pipeline | Sim | Sim | Não |
| Taxa de conversão por estado (funil) | Sim | Sim | Não |
| Contatos por papel e status | Sim | Sim | Sim (próprios) |
| Atividades registradas por período | Sim | Sim (equipe) | Sim (próprias) |
| Casos cancelados com motivo | Sim | Sim (equipe) | Sim (próprios) |
| Log de auditoria | Sim | Não | Não |
| Exportação de dados pessoais (LGPD) | Sim | Não | Não |

- Bloquear acesso a relatórios não autorizados — os itens correspondentes não aparecem na navegação para papéis sem acesso.

**Critério de aceite:** Analista RS não vê "Desempenho por Analista RS" na navegação. Coordenador RS consegue abrir "Comissões estimadas no pipeline" mas não vê "Comissões confirmadas". Admin RS acessa todos os 10 relatórios.

---

### RF-CRM-072 — Exportação de dados com log obrigatório
**Origem:** RN-151
**Papel:** Admin RS (dados pessoais), Coordenador RS (dados agregados)

O sistema deve:
- Disponibilizar exportação de relatórios nos formatos CSV e XLSX.
- Para exportações com dados pessoais (CPF, e-mail, telefone de Cedentes/Cessionários): restringir ao Admin RS e registrar obrigatoriamente no log de exportações: usuário, data/hora, filtros utilizados e quantidade de registros exportados.
- Para exportações sem dados pessoais (métricas, valores agregados, identificadores de Caso): disponibilizar para Coordenador RS conforme matrix RF-CRM-071.
- Log de exportações: auditável pelo Admin RS, não editável nem excluível.

**Critério de aceite:** Coordenador RS consegue exportar relatório de funil de conversão (sem dados pessoais). Tentativa de Coordenador RS exportar relatório de exportação de dados pessoais é bloqueada. Toda exportação pelo Admin RS com dados pessoais aparece no log com todos os campos obrigatórios.

---

### RF-CRM-073 — Relatório de inteligência de mercado
**Origem:** RN-152
**Papel:** Admin RS

O sistema deve:
- Exibir em "Relatórios → Inteligência de Mercado" análise agregada e anonimizada contendo:
  1. Distribuição de Δ por empreendimento e região.
  2. Tempo médio de Match por faixa de valor.
  3. Taxa de Anuência negada por incorporadora.
  4. Perfil dos Cessionários recorrentes (faixa de valor, tipologia, região).
  5. Sazonalidade de abertura de Casos (mês a mês, últimos 24 meses).
- Aplicar anonimização: nenhum dado pessoal exibido. Agrupamentos com menos de 5 Casos mascarados para evitar reidentificação.

**Critério de aceite:** Relatório de Inteligência de Mercado não acessível pelo Coordenador RS ou Analista RS. Agrupamento com 3 Casos exibe "N/A" ou "< 5 casos" em vez dos dados. Nenhum CPF, nome ou e-mail aparece no relatório.
