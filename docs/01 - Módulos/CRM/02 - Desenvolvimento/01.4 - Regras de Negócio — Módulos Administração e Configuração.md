> 📌 **TL;DR**
>
> - **4 módulos administrativos documentados:** Equipe · Dashboard Executivo · Configurações do Sistema · Relatórios e Exportações.
> - **Módulo Equipe** governa criação, edição, suspensão e desligamento de usuários internos — responsabilidade exclusiva do Admin RS.
> - **Dashboard Executivo** oferece visão consolidada de pipeline, receita e desempenho para Coordenador RS e Admin RS.

> - **Regra inegociável:** toda alteração em Configurações do Sistema gera registro de auditoria com usuário, data/hora, campo alterado e valores anterior/posterior.

## 👤 17. Módulo: Equipe

**Objeto principal:** Usuário Interno
**Estados:** Ativo · Suspenso · Desligado
**Operações:** Convidar · Ativar · Editar papel · Suspender · Reativar · Desligar · Transferir Casos

---

**RN-139: Gestão do ciclo de vida de usuários internos**

1. O Admin RS é o único responsável pela criação, edição de papel, suspensão e desligamento de usuários internos.
2. **Estados possíveis:**
   - **Ativo:** usuário com acesso pleno conforme seu papel.
   - **Suspenso:** acesso temporariamente bloqueado (ex: férias, afastamento). Os Casos permanecem atribuídos ao usuário suspenso — redistribuição é opcional.
   - **Desligado:** acesso permanentemente revogado. Todos os Casos atribuídos devem ser redistribuídos antes do desligamento.
3. **Transições permitidas:**

   ```mermaid
   stateDiagram-v2
       [*] --> Ativo : convite aceito
       Ativo --> Suspenso : Admin suspende
       Suspenso --> Ativo : Admin reativa
       Ativo --> Desligado : Admin desliga
       Suspenso --> Desligado : Admin desliga
       Desligado --> [*]
   ```

4. **Consequência se violada:** usuários desligados com acesso ativo comprometem a segurança e a integridade dos dados de Cedentes e Cessionários.

---

**RN-140: Suspensão de usuário interno**

1. O Admin RS acessa "Equipe" → perfil do usuário → "Suspender acesso".
2. O sistema exige motivo da suspensão (campo selecionável: Férias · Afastamento médico · Investigação interna · Outros).
3. **Se o usuário tem Casos ativos:** o sistema exibe lista dos Casos e pergunta: "Deseja redistribuir os Casos agora ou manter atribuídos ao usuário suspenso?" O Admin RS escolhe.
4. **Após a suspensão:** o usuário não consegue logar e recebe e-mail informativo: "Seu acesso ao CRM foi temporariamente suspenso. Entre em contato com seu gestor para mais informações."
5. **Sessões ativas do usuário são encerradas imediatamente** após a suspensão.
6. **Consequência se violada:** usuário suspenso sem revogação de sessão ativa mantém acesso durante a sessão corrente, criando janela de vulnerabilidade.

---

**RN-141: Desligamento de usuário interno**

1. O Admin RS acessa "Equipe" → perfil do usuário → "Desligar usuário".
2. **Pré-requisito obrigatório:** todos os Casos atribuídos ao usuário devem ser redistribuídos antes de confirmar o desligamento. O sistema bloqueia o desligamento se houver Casos ativos não redistribuídos.
3. **Se todos os Casos foram redistribuídos:** o sistema confirma o desligamento, revoga o acesso permanentemente e encerra todas as sessões ativas.
4. **Dados do usuário desligado são preservados no histórico** (Atividades registradas, Casos que conduziu, linha do tempo) — nunca excluídos. O nome é exibido como "[Nome] (Desligado)" nos registros históricos.
5. **O e-mail do usuário desligado não pode ser reutilizado** para criar uma nova conta no CRM por 180 dias. [DECISÃO AUTÔNOMA — 180 dias: janela de segurança para evitar reutilização imediata de credencial em caso de desligamento por motivo disciplinar.]
6. **Consequência se violada:** histórico de Casos sem vínculo ao Analista responsável compromete auditoria e impossibilita análise de desempenho histórico.

---

**RN-142: Alteração de papel de usuário interno**

1. O Admin RS pode alterar o papel de um usuário entre Analista RS e Coordenador RS.
2. **Ao rebaixar de Coordenador RS para Analista RS:** o sistema verifica se o usuário tem Casos de outros Analistas sob sua supervisão e exibe alerta para redistribuição.
3. **Ao promover de Analista RS para Coordenador RS:** o usuário passa a ter acesso imediato a todos os Casos da equipe e ao Dashboard consolidado.
4. **A alteração de papel é registrada no log de auditoria** com data, responsável e papéis anterior/posterior.
5. **Nenhum usuário pode ter papel Admin RS** além do(s) Admin(s) previamente designados. A criação de novos Admins RS requer aprovação de outro Admin RS existente (quatro olhos). [DECISÃO AUTÔNOMA — quatro olhos para Admin: padrão de governança SaaS B2B interno para evitar escalada de privilégios unilateral.]
6. **Consequência se violada:** alteração de papel sem governança cria brechas de acesso indevido a dados confidenciais de Casos e Configurações.

---

**RN-143: Relatório de carga da equipe**

1. O Coordenador RS e o Admin RS acessam "Equipe → Carga atual" para visualizar a distribuição de Casos entre os Analistas RS.
2. **O relatório exibe por Analista RS:**
   - 2.1. Total de Casos ativos por estado.
   - 2.2. Casos com SLA em risco (amarelo) e vencido (vermelho).
   - 2.3. Follow-ups vencidos sem resolução.
   - 2.4. Tempo médio de conclusão dos últimos 10 Casos fechados.
3. **Usado como base para redistribuição** (conforme RN-041, Parte 01.2) e para decisões de contratação.
4. **Consequência se violada:** sem visibilidade de carga, a gestão distribui Casos de forma desequilibrada, sobrecarregando Analistas e comprometendo SLAs.

## 📊 18. Módulo: Dashboard Executivo

**Objeto principal:** Painel executivo (somente leitura)
**Operações:** Visualizar · Filtrar por período · Exportar recorte do Dashboard

---

**RN-144: Visão executiva do Pipeline**

1. O Coordenador RS e o Admin RS acessam o Dashboard Executivo no painel principal.
2. **Métricas exibidas em tempo real (≤5 min de defasagem):**

   | **Métrica** | **Descrição** |
   |---|---|
   | Casos abertos | Total por estado e por Analista RS |
   | Taxa de conversão por estado | % de Casos que avançam de cada estado |
   | Ciclo médio total | Dias corridos da abertura ao Fechamento (últimos 90 dias) |
   | Ciclo médio por estado | Dias úteis médios em cada estado (últimos 90 dias) |
   | Casos em risco de SLA | Alertas amarelos e vermelhos ativos |
   | Previsão de Fechamentos | Casos em Formalização/Anuência com data estimada |
   | Taxa de cancelamento | % de Casos cancelados por motivo (últimos 90 dias) |

3. **O Analista RS não tem acesso ao Dashboard Executivo** — apenas ao painel pessoal (Casos próprios, agenda de Follow-ups, alertas de SLA).
4. **Consequência se violada:** visibilidade do Dashboard por Analistas RS pode expor estratégias de negociação de outros membros da equipe.

---

**RN-145: Métricas financeiras no Dashboard**

1. O Admin RS acessa a aba "Financeiro" do Dashboard com as seguintes métricas:

   | **Métrica** | **Descrição** |
   |---|---|
   | Receita confirmada no mês | Soma das comissões de Casos Concluídos no mês corrente |
   | Receita estimada no pipeline | Soma das comissões estimadas de Casos em Negociação/Anuência/Formalização |
   | Ticket médio por Caso | Média de comissão total (cedente + cessionário) dos últimos 12 meses |
   | Margem de contribuição estimada | Receita confirmada − custo incremental estimado por Caso |
   | Descontos concedidos no mês | Soma e percentual médio de descontos aplicados (RN-052, Parte 01.2) |
   | Casos no breakeven (≥3/mês) | Indicador binário: mês corrente atingiu ou não o breakeven operacional |

2. **A aba "Financeiro" não está disponível para o Coordenador RS** — apenas para o Admin RS. [DECISÃO AUTÔNOMA — financeiro restrito ao Admin: dados de margem e breakeven são estratégicos; Coordenador RS opera com KPIs operacionais, não financeiros.]
3. **Consequência se violada:** exposição de dados de margem e breakeven a usuários sem necessidade operacional cria risco de vazamento de informação estratégica.

---

**RN-146: Análise de funil de conversão**

1. O Coordenador RS e o Admin RS acessam "Dashboard → Funil" para visualizar a taxa de conversão entre estados.
2. **O funil exibe:** para cada par de estados consecutivos, o percentual de Casos que avançaram vs. percentual que cancelaram, com comparativo entre o mês corrente e a média histórica dos últimos 90 dias.
3. **Estados com taxa de cancelamento acima de 20%** são destacados em vermelho como "Gargalo identificado". [DECISÃO AUTÔNOMA — 20%: limiar de gargalo razoável para operação imobiliária com ciclo de 45–60 dias; abaixo de 20% é variação normal do mercado.]
4. **Drilling down:** ao clicar em qualquer estado do funil, o sistema exibe a lista dos Casos que cancelaram naquele estado com os respectivos motivos.
5. **Consequência se violada:** sem análise de funil, a equipe não identifica onde o processo está quebrando — melhorias são baseadas em percepção, não em dados.

## ⚙️ 19. Módulo: Configurações do Sistema

**Objeto principal:** Parâmetro de Configuração
**Operações:** Visualizar · Editar · Auditar histórico de alterações

> 🔴 **Regra crítica:** toda alteração em Configurações do Sistema é auditada automaticamente — usuário, campo, valor anterior, valor novo, data/hora. Nenhuma alteração pode ser feita sem registro.

---

**RN-147: Parâmetros configuráveis do CRM**

| **Parâmetro** | **Padrão** | **Quem pode alterar** | **Impacto** |
|---|---|---|---|
| SLA por estado (dias úteis) | Tabela 5.1 (Parte 01.1) | Admin RS | Alertas automáticos de SLA |
| Limite de Casos por Analista RS | 30 | Admin RS | Alerta de sobrecarga na redistribuição |
| Prazo de validade de proposta (horas) | 48h | Admin RS | Expiração automática de propostas |
| Prazo de convite de Parceiro Externo (dias) | 7 dias | Admin RS | Expiração de links de convite |
| Prazo de arquivamento de Contato inativo (meses) | 12 meses | Admin RS | Arquivamento automático de Contatos |
| Prazo de arquivamento de Caso cancelado (dias) | 90 dias | Admin RS | Arquivamento automático de Casos |
| Tabela de limites de desconto | Tabela RN-052 (Parte 01.2) | Admin RS | Limite de desconto por faixa |
| Ciclo total alerta (dias corridos) | 60 dias | Admin RS | Alerta de Caso em risco |
| Follow-up: prazo de aviso (minutos antes) | 30 min | Admin RS | Notificação de Follow-up |
| Inatividade de sessão (minutos) | 60 min | Admin RS | Encerramento automático de sessão |
| Retenção de dados — Casos cancelados (anos) | 5 anos | Admin RS | Exclusão/anonimização automática |
| Retenção de dados — Casos concluídos (anos) | 10 anos | Admin RS | Exclusão/anonimização automática |
| Hora de disparo do relatório semanal | Domingo 20h | Admin RS | E-mail automático de desempenho |
| Defasagem máxima de atividade retroativa (dias) | 30 dias | Admin RS | Limite de retroatividade de Atividades |

---

**RN-148: Processo de alteração de parâmetro crítico**

1. O Admin RS acessa "Configurações → Parâmetros" e edita um parâmetro.
2. **Para parâmetros classificados como críticos** (SLA por estado, limites de desconto, retenção de dados e limite de Casos por Analista), o sistema exige confirmação de segundo Admin RS (quatro olhos).
3. **Se há apenas 1 Admin RS ativo:** o sistema exibe aviso: "Este parâmetro requer aprovação de dois Admins. O sistema não tem um segundo Admin ativo. A alteração fica pendente de aprovação." O parâmetro permanece com o valor anterior até a aprovação.
4. **Após aprovação:** a alteração é aplicada imediatamente e registrada no log de auditoria.
5. **O log de auditoria de Configurações** é acessível apenas pelo Admin RS e não pode ser editado ou excluído.
6. **Consequência se violada:** alterações unilaterais de SLA ou limites de desconto sem auditoria comprometem a governança operacional e financeira.

---

**RN-149: Gestão de templates de mensagem**

1. O Admin RS cria, edita ou desativa templates de mensagem (conforme RN-104, Parte 01.3).
2. **Templates ativos** ficam disponíveis para todos os Analistas RS.
3. **Templates desativados** não aparecem para Analistas RS, mas mantêm-se no histórico para auditoria.
4. **Toda edição de template** gera nova versão (v1.0, v1.1, v2.0). A versão anterior é arquivada — nunca excluída.
5. **Consequência se violada:** templates desatualizados em uso podem gerar comunicações com informações incorretas de prazo, valor ou processo, comprometendo a relação com Cedentes e Cessionários.

## 📤 20. Módulo: Relatórios e Exportações

**Objeto principal:** Relatório exportado
**Operações:** Gerar · Filtrar · Exportar · Auditar exportações

---

**RN-150: Relatórios disponíveis por papel**

| **Relatório** | **Admin RS** | **Coordenador RS** | **Analista RS** |
|---|---|---|---|
| Pipeline completo (todos os estados) | ✅ | ✅ Da equipe | ✅ Próprios Casos |
| Desempenho por Analista RS | ✅ | ✅ Da equipe | ❌ |
| Comissões confirmadas (período) | ✅ | ❌ | ❌ |
| Comissões estimadas no pipeline | ✅ | ✅ | ❌ |
| Taxa de conversão por estado (funil) | ✅ | ✅ | ❌ |
| Contatos por papel e status | ✅ | ✅ | ✅ Próprios |
| Atividades registradas por período | ✅ | ✅ Da equipe | ✅ Próprias |
| Casos cancelados com motivo | ✅ | ✅ Da equipe | ✅ Próprios |
| Log de auditoria | ✅ | ❌ | ❌ |
| Exportação de dados pessoais (LGPD) | ✅ | ❌ | ❌ |

---

**RN-151: Exportação de dados**

1. O usuário autorizado acessa "Relatórios → Exportar" e configura o relatório (período, filtros, campos desejados).
2. **Formatos disponíveis para exportação:** CSV e XLSX.
3. **Exportações com dados pessoais** (CPF, e-mail, telefone de Cedentes/Cessionários) são restritas ao Admin RS e registradas no log de exportações com: usuário, data/hora, filtros utilizados e quantidade de registros exportados.
4. **Exportações sem dados pessoais** (métricas, valores agregados, identificadores de Caso) estão disponíveis para o Coordenador RS conforme tabela RN-150.
5. **O log de exportações é auditável** pelo Admin RS e não pode ser editado ou excluído.
6. **Consequência se violada:** exportações de dados pessoais sem rastreamento criam risco de vazamento não detectado, violando o princípio de prestação de contas da LGPD.

---

**RN-152: Relatório de inteligência de mercado**

1. O Admin RS acessa "Relatórios → Inteligência de Mercado" para análise agregada e anonimizada.
2. **O relatório contém:**
   - 2.1. Distribuição de Δ por empreendimento e região.
   - 2.2. Tempo médio de Match por faixa de valor.
   - 2.3. Taxa de Anuência negada por incorporadora.
   - 2.4. Perfil dos Cessionários recorrentes (faixa de valor, tipologia, região).
   - 2.5. Sazonalidade de abertura de Casos (mês a mês, últimos 24 meses).
3. **Os dados são anonimizados:** nenhum dado pessoal é exibido. Agrupamentos com menos de 5 Casos são mascarados para evitar reidentificação. [DECISÃO AUTÔNOMA — mínimo de 5 para anonimização: padrão de k-anonimidade aplicado a datasets pequenos.]
4. **Consequência se violada:** relatórios de inteligência com dados pessoais identificáveis configuram uso secundário de dados sem base legal adequada na LGPD.
